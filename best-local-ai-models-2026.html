Advanced: Model Quantization Explained

What is quantization?

Reducing model precision from 16-bit (FP16) to 4-bit or 8-bit integers. Result: 2-4x smaller files, 2-3x faster inference, minimal quality loss.

Quantization formats:

Format Size vs Original Quality Loss Speed Gain Recommended For
FP16 (full) 100% 0% Baseline 80GB+ VRAM
Q8_0 50% <1% 1.5x 48GB VRAM
Q5_K_M 35% <2% 2x 24GB VRAM
Q4_K_M 25% 3-5% 2.5x 12GB VRAM
Q3_K_M 20% 8-10% 3x 8GB VRAM

Best practice: Use Q4_K_M or Q5_K_M. Quality loss is negligible, speed gain is significant.

Ollama auto-downloads optimal quantization based on your VRAM.

Troubleshooting & Optimization

Issue 1: Model Runs Slowly (< 10 tokens/sec)

Causes: - CPU-only mode (no GPU detected) - Insufficient VRAM (model offloaded to RAM) - Background processes using GPU

Fixes: 1. Check GPU detection: ollama ps (should show GPU usage) 2. Use smaller model or more aggressive quantization 3. Close other GPU-heavy apps (browsers, games) 4. Enable GPU acceleration in LM Studio settings

Issue 2: Out of Memory Errors

Causes: - Model too large for available VRAM/RAM - Other apps consuming memory

Fixes: 1. Use more aggressive quantization (Q4 instead of Q5) 2. Choose smaller model (8B instead of 70B) 3. Increase system swap (Linux/macOS) 4. Close memory-heavy apps

Issue 3: Poor Response Quality

Causes: - Over-quantization (Q3 or lower) - Inappropriate model for task - Bad system prompt

Fixes: 1. Use Q4_K_M or higher quantization 2. Switch to task-specific model (Qwen for coding, DeepSeek for reasoning) 3. Refine system prompt with examples

Issue 4: Context Window Errors

Causes: - Conversation too long for model's context - Large document upload

Fixes: 1. Use model with larger context (Llama 3.3: 128K tokens) 2. Summarize earlier parts of conversation 3. Split large documents into chunks
